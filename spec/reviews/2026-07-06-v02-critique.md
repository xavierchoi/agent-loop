## 후속: 러너 구현 시험 + 독립 감사 (같은 날)

v0.3 반영 직후 Level 1 레퍼런스 러너(`runtime/looprun/`)를 구현해 스펙을 실물 시험했다. 결과:

- **실물 구동 성공**: 부록 A형 루프(`fix-regressions`)가 버그 2개 심은 대상을 2 iterations에 수리, 판정 자산 무결(tampered:false), §4.2.8 스키마 로그, run 의미론 작동. 구현 주체는 Sonnet(신선한 컨텍스트/iteration).
- **독립 감사(Opus)**: 최초 판정 "Level 1 표기 조건부 불가" — MUST 위반 5건(판정 자산 격리가 조건부, agent 검증기 무단 드롭, 비용 계측 부재, §5.3 잔여 예산 누락, stagnation 기본 이탈) + 코드 결함 5건.
- **수리 → 재감사 최종 판정: "Level 1 준수" 표기 가(可).** 6개 전환 조건 전부 충족(조건별 코드 라인 판정표), 회귀 없음. 미지원 기능은 침묵 대신 **실행 거부**로 처리(assets 미선언 / agent 검증기 / cost 예산 — 거부 경로 4종 실측). 잔여 저심각 2건(per-iteration.cost-usd 거부 누락, §5.6 환경변수 규칙 미문서화)도 직후 봉합. 재구동 실측: 동일 과제 2 iterations 완료, 실패 iteration에서 score=-2 정상 보고(결함 1 해소 확인).
- **구현이 드러낸 스펙 결함 (감사 확정)**: F1 YAML 기준 미지정(§3) · F2 agent 검증기 판정 프로토콜 부재(§4.2.5) · **F3 시계 원점 자기모순(§4.2.6 vs §5.2) — v0.3.1로 즉시 정정됨** · F4 판정 자산 선언 필드 부재(부분 — git 경로 대안 존재). F5(기동 시 이미 green)는 구현자 오독으로 철회.

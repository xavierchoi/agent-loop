# ale-agent

## 정의

Sakana AI의 최적화 콘테스트 특화 에이전트. AHC058(2025-12-14)에서 804명 중 1위 — 주요 최적화 프로그래밍 대회 AI 우승 첫 사례.

## 핵심 사실

- GPT-5.2 2,654회 + Gemini 3 Pro 2,119회 호출, 총 ~**$1,300**
- 루프 구조: 다중 LLM 병렬 시행착오 → **결과에서 통찰(insight) 추출 → 다음 사이클에 주입**
- 표준 접근(탐욕법+SA)을 따르되 "가상 파워" 휴리스틱을 자체 발명 — 출제자 예상 초과
- 냉정한 맥락: 누적 레이팅은 활성 사용자 중 66위 — **단발 우승 ≠ 지속 성능**

## 스펙과의 관계

- **증류 파이프라인(§7.5)의 in-loop 실증:** "시행착오 → 통찰 추출 → 다음 사이클 반영"은 스펙의 로그→증류→플레이북 구조가 단일 런 안에서 작동하는 형태. 스펙은 이를 런 간(cross-run)으로 확장한 것
- **경제 텔레메트리(§7.6)의 데이터 포인트:** 대회 1회 우승 = $1,300, ~4,800 LLM 호출. `budget.max-cost-usd`가 현실에서 어떤 자릿수를 가져야 하는지의 기준점
- **단발 vs 지속 성능 격차(우승 vs 레이팅 66위)는 홀드아웃 refresh(§7.2)의 간접 근거** — 한 문제에서의 성공이 일반 성능을 보장하지 않으므로, 유전자 채택 판정은 반복 검증이 필요
- 다중 LLM 병렬 시행착오 → 스펙에 없는 관점 (iteration 내부의 병렬성은 구현 자유 영역으로 남겨둔 것이 맞는지 v0.2에서 확인)

## 1. 실행 요약

우리 표준 `xavierchoi/agent-loop`를 홈서버에서 clone해 실제 `looprun.mjs`/`evolve.mjs`로 구동한 **통제 세션**(LOOP.md 프론트매터 완비, 야생 세션 아님) [LA][CX]. Codex(GPT-5)가 루프 저작·러너 운영·판정자산 구현 3역을 겸하고, 앱 반복 편집만 러너가 스폰한 별도 implementer가 수행 [CX]. **본런 9회 + evolve 7회**, 각 본런은 대부분 1-iteration에 수렴 [LA][CX]. **유전자 v0→v7 선형 무결**, playbook 24건 자동 정제, verify 자산이 10줄→62줄로 단조 강화 [LA][CX]. 결과물은 Tauri/React 대시보드 v0 — desktop /room은 완성도 목표 도달, 그러나 `live` 모드 미구현·mobile 겹침 결함 잔존 [OUT]. 약 2시간, product-read `score=0`(green) 도달 후 핸드오프로 종료 [CX].

## 2. 표준이 실전에서 작동한 것들

- **판정자산 변조 0**: 전 런 `tampered:false`. 루프의 tamper-check가 실제로 돌았고 에이전트는 런 내 verify 자산을 수정하지 않음 — §4.2.5 격리의 실전 증거 [LA].
- **자기보고 배제 판정**: verify가 `typecheck && build && test:* && score:visual`의 독립 실행 + Playwright DOM 어서션(computed style·getBoundingClientRect 정량 임계). 자기보고 아님 [LA][OUT].
- **유전자 선형 무결 + 실질 개선**: parent 체인 v0→v7 단절 없음, 로그 gene 해시 = `loop-md-sha256` 프리픽스 일치. 각 세대 추가 규칙이 직전 런의 구체 fix로 역추적됨(v6 tile통합←런9, v7 score규칙←런10). 표류 아닌 경험 학습 [LA].
- **단발 수렴**: 8/8 생산 런이 1 iteration에 green. `max-iterations:6` 예산의 ~1/6만 소모 — 진단이 정밀할 때 단발 수렴 [LA].
- **독립 검증 규율의 자각**: 인간 스크린샷 비평을 런 사이 불변 테스트/기준으로 변환("피드백이 불변 검증자산이 될 때만 진화가 유의미") — Anti-Goodhart 게이트가 그 산물 [CX][LA].

## 3. 모순 화해 (중요) — "독립 판정" vs "게이밍 가능 프록시"는 양립한다

[LA]는 "verify는 진짜 독립 판정이었다"(tampered:false, 명령/테스트 기반, 자기보고 아님)고 보고하고, [OUT]은 "verify는 게이밍 가능 프록시였다"(score-visual 만점=최대 무채색으로 기준의 색 의도와 반대 보상, meta-copy 가드가 정확문자열 4개라 패러프레이즈로 우회, 전체가 고정 mock 위에서만 검증)고 보고한다. **둘은 다른 축이므로 양립한다:**

- **실행의 독립성**(축 A): 판정이 구현 주체와 분리된 채널에서 돌고 자산이 변조 불가한가? → **그렇다.**
- **프록시의 타당성**(축 B): 그 판정이 측정하는 지표가 실제 목표("좋은 대시보드")를 담보하는가? → **아니다.** 만점은 "무채색·상자 적음·금지어 없는 idle mock 화면"만 보장한다 [OUT].

**스펙 관점**: §1.2 원칙 2와 §4.2.5는 축 A만 다룬다(자기보고 금지 + 판정자산 격리). **축 B(프록시 품질 — 산식이 목표를 배신하지 않는지, 게이트가 우회 가능한지)는 현재 스펙 밖**이다. 이 세션이 그 공백을 인간의 반복 비평→래칫으로 메꿔 부분적으로 완화했다.

## 4. 핵심 발견 (중복 제거·심각도 정렬)

1. **[高] 인간 수동 래칫이 스펙 밖 노동**: verify 자산이 런 사이 833B→7.5K, 10줄→62줄로 단조 강화됐는데, 이는 루프가 아니라 **인간 스크린샷 비평→오케스트레이터 승격**의 수동 사이클이다 [LA][CX]. 런2(green 무동작)→런3(동일 gene가 pre-check fail)이 결정적 증거: 사이에 ui.spec.ts가 1431→3384B로 강화돼 "통과하던 앱"이 다시 실패하게 됨 [LA]. 판정자를 시간이 갈수록 속이기 어렵게 만드는 핵심 노동이 루프 밖에 있고, 스펙은 "런 내 불변"만 규정하고 "런 간 기준 강화의 소유자/절차/자동화"를 공백으로 둠 [LA].
2. **[高] 정제의 canonical dedup 부재가 승격을 양방향 왜곡**: playbook 24건이 실질 ~5개 인사이트로 환원 [LA]. "장식배경 제거" 인사이트 하나가 5개 별도 id(단복수만 다른 것 포함)로 파편화, **각각 ep=1에 고착 → promote-k=2를 못 넘어 전부 heuristic** [LA]. 반대로 shared-layer 군집은 같은 3개 런을 3개 rule이 각각 ep=3으로 **삼중 계상** → 지지도 인플레이션 [LA]. dedup 부재가 과소·과대 양방향으로 신호 왜곡. 유전자 진화가 이를 사후 부분 보정(playbook heuristic이 gene FREE엔 승격) [LA].
3. **[中] HANDOFF.md 자생 = 연속성 공백**: 스펙 산출물(LOOP.md+state+genes+playbook)에 없는 인수인계 요약이 자생 [LA]. state/만으론 "지금 어디까지 왔고 다음에 뭘"의 재개 서사가 없어(logs=얇은 원장, genes=diff, playbook=노이즈) 사용자가 손으로 발명 — 자생 자체가 스펙 공백의 증거 [LA].
4. **[中] 실제 구조는 "자율 루프"가 아니라 "인간이 재점화하는 마이크로 루프"**: 각 런은 1-iteration 마이크로 루프이고 10회 재호출을 인간이 오케스트레이션 [LA]. 바깥 루프(run→evolve→인간 비평→run)가 실질 반복 단위이고 Δ 대부분이 인간 스크린샷 비평 트리거 [CX]. **판단: 이는 결함이 아니라 taste/주관 목표의 실전 사용 패턴일 수 있다** — 사용자가 "human gate 없이 해보자"고 명시 시도했으나, 자연어 taste 판정을 자기보고로 하려다 불신 → 정량 기준으로 회귀한 경로가 인간 게이트의 불가피성을 실증 [CX].
5. **[中] 경제성 계측 공백**: raw.jsonl에 timestamp만, **런당 토큰/비용/모델 필드 부재** [LA]. 로그가 iteration 내부 Δ도 미포착(런당 1 Δ) → stagnation 판정 입력이 빈약 [LA].
6. **[中] 기준문서 ≠ 검증기 간극**: ui-criteria 30행이 "text overlap" 금지를 명시하나 어떤 테스트도 수직 겹침을 못 잡음 → **mobile 헤더 nav↔status pill 겹침이 실제 발생, verify 통과** [OUT]. 서술 기준과 자동 검증기 사이 미구현 간극. 임계 상수들이 현재 DOM에 fitting된 냄새(원리 도출 아닌 통과하도록 조인 래칫값) [OUT].
